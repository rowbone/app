Sweet Alert

操作提示框
Based by sweet-alert and ngSweetAlert
使用说明
  1. controller 注入服务 OperationService
  2. 参数说明
    • confirmTitle: '', // 调用后台接口前的提示信息
    • confirmText: '',
    • successTitle: '', // 操作成功时的提示信息
    • successText: '',
    • errorTitle: '', // 操作失败时的提示信息
    • errorText: '',
    • actionUrl: '', // 操作对应的后台接口
    • data: {}, // 提交到后台的数据
    • isSuccessBack: true, // 操作成功是否返回
    • redirectState: 'app.home' // 操作成功的重定向地址(state跳转，isSuccessBack为false时起作用)
    • successFunc: func, // 操作成功的回调方法
    • errorFunc: func // 操作失败的回调方法
  3. 示例
    1. 调用接口前有提示，调用之后有提示，操作成功后停在当前页面 
      OperationService.interAction({
  'actionUrl': '/home',
  'data': { 'name': 'abc', 'age': 8 },
  'isSuccessBack': false
});
    2. 直接调用后台接口，成功不返回 
      OperationService.interAction({
	'showConfirm': false,
  'actionUrl': '/home',
  'isSuccessBack': false
});
    3. 操作成功返回[isSuccessBack 为 true时，redirectState 参数不起作用] 
      OperationService.interAction({
  'actionUrl': '/home',
  'isSuccessBack': true,
  'redirectState': 'app.home.contactlist2'
});
    4. 操作成功后重定向 
      OperationService.interAction({
  'actionUrl': '/home',
  'isSuccessBack': false,
  'redirectState': 'app.home.contactlist2'
});
    5. 提示框

      OperationService.alertAction('这里是提示信息');

      或者 OperationService.alertAction({alertMsg: '这里是提示信息', type: 'info'});其中type的可选项为infowarningsuccesserror

    6. 日程：[是否更新所有相同日程] 
      $scope.alertHtml = function() {
  var callbackFunc = function(data) {
    console.log('alertHtml callbackFunc-->', data);
  };

  var params = {
    yesConfirmFunc: function() {console.log('yes..........')},
    noConfirmFunc: function() {console.log('no..........')},
    callback: callbackFunc,
    yesUrl: '/home',
    noUrl: '/error',
    confirmTitle: '这里是操作提示！',
    isSuccessBack: 'false',
    redirectState: 'app.components.stepper'
  };
  OperationService.alertHtml(params);
};
      参数 说明
      yesUrl 选择"是"所对应的后台接口
      noUrl 选择"否"所对应的后台接口
      yesConfirmFunc 选择"是"所对应的后台接口
      noConfirmFunc 选择"否"所对应的后台接口
      yesData yesUrl对应提交到后台接口的参数
      noData noUrl对应提交到后台接口的参数
适用场景
  1. 页面的保存、删除操作
  2. 涉及工作流的提交操作的调用方法： workflowService.submitWorkflow("expenseLabor",$scope.paramWorkflow,"")，中的第三个参数为空，则操作成功会直接返回（一般是返回列表），如果给定重定向地址state，则会重定向
